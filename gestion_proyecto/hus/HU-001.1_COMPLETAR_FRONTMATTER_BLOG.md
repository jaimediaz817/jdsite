## 🎯 Objetivo
Completar el sistema de importación de blogs para soportar TODOS los campos del frontmatter que ya estamos usando, incluyendo la imagen de portada.
Ningún campo que escribamos en el markdown se pierda. Todo se importe automaticamente.

---

## ✅ Criterios de Aceptación

1.  [x] **ESTANDAR OFICIAL FRONTMATTER DEFINITIVO ✅ ESTABLECIDO:**
    ```yaml
    ---
    title: "Titulo del blog"
    description: "Descripcion corta para listados y SEO"
    date: 2026-04-26
    draft: false
    image: "cover.jpg"
    author: "Jaime Díaz"
    category: "UI/UX"
    tags: ["UI", "UX", "Diseño Web"]
    meta_title: "Titulo SEO especifico"
    meta_description: "Descripcion SEO para Google y redes"
    ---
    ```

    ✅ **REGLA OFICIAL:**
    > TODOS los campos string VAN CON COMILLAS DOBLES.
    > El comando se encarga 100% de limpiarlas automaticamente.
    > En base de datos NUNCA se guardan las comillas.

2.  [ ] **Todos los campos se importan automaticamente:**
    - [ ] `title` → ya funciona ✅
    - [ ] `description` → se guarda en el campo correspondiente del modelo
    - [ ] `date` → se usa como `publish_date` NO fecha actual
    - [ ] `draft` → si `true` se guarda como `is_published = False`
    - [ ] `image` → ✅ IMAGEN DE PORTADA
    - [ ] `category` → ya funciona ✅
    - [ ] `tags` → ya funciona ✅
    - [ ] `meta_title` → se guarda en el campo SEO
    - [ ] `meta_description` → se guarda en el campo SEO

3.  [x] **✅ FUNCIONALIDAD ESPECIAL PARA IMAGEN DE PORTADA:**
    ✅ **SOLUCION A TU PROBLEMA EXACTO:**
    - Puedes pegar la imagen con Ctrl+V NORMALMENTE dentro del markdown como si fuera cualquier otra imagen
    - El comando detecta AUTOMATICAMENTE que es la portada
    - ✅ **NO APARECE DENTRO DEL CONTENIDO DEL BLOG**
    - ✅ Solo se usa como imagen de portada en listados, redes, etc
    - ✅ No tienes que borrarla ni moverla ni hacer nada
    - ✅ Simplemente la pegas en el principio del markdown y ya

    ✅ **Como funciona:**
    1.  Tu escribes al principio del blog:
        ```markdown
        # Titulo del blog
        ![alt text](cover.jpg)
        
        Empieza el contenido...
        ```

    2.  El comando detecta que es la PRIMERA imagen del articulo
    3.  La extrae automaticamente
    4.  La asigna como `cover_image` del blog
    5.  La ELIMINA del contenido html final
    6.  ✅ No aparece en el detalle del blog, solo como portada

    ✅ 100% transparente para ti. Tu solo pegas la imagen como siempre. El comando se encarga de todo el resto.

4.  [ ] **Retrocompatibilidad 100%:**
    - ✅ Si un campo no existe en el frontmatter, usa valor por defecto
    - ✅ Todos los blogs existentes siguen funcionando exactamente igual
    - ✅ El comando se puede ejecutar sin romper nada
