## 🎯 Como visitante del blog
**Quiero** poder dar reacciones a los comentarios de los artículos sin tener que registrarme ni dar ningún dato personal
**Para** expresar mi opinión sobre las opiniones de otros usuarios de forma rápida y sin fricción

---

## 📊 Contexto técnico
- App `reactions` separada de app `blog` por diseño arquitectónico
- Sistema de reacciones a artículos YA implementado y funcionando
- BlogComment existe en `blog.models` con soporte para respuestas anidadas
- NO se modifica BlogReaction existente

---

## ✅ CRITERIOS DE ACEPTACIÓN

### Funcionales
1. 🔴 No se pide absolutamente ningún dato personal al usuario
2. 🔴 Una misma IP solo puede dar una vez cada tipo de reacción por comentario
3. 🔴 Si el usuario presiona nuevamente la misma reacción, esta se quita
4. 🔴 SOLO UNA REACCIÓN por usuario por comentario
5. 🔴 El sistema funciona completamente incluso con Javascript desactivado
5. 🟡 Se muestran contadores actualizados en tiempo real
6. 🟡 No se usa ninguna cookie ni storage del navegador
7. 🟡 Robots y crawlers no pueden registrar reacciones

### Usabilidad
8. 🔴 Todas las interacciones tienen animaciones profesionales y naturales
9. 🔴 El feedback visual es inmediato antes de que termine la petición al servidor
10. 🔴 Área de click mínimo 48x48px por cada reacción
11. 🟡 No hay animaciones de más de 300ms
12. 🟡 No hay modales, confirmaciones ni interrupciones
13. 🟡 UI de reacciones similar a la de artículos para consistencia visual

### Técnicos
14. 🔴 No se rompe nada del funcionamiento actual del blog
15. 🔴 No se modifica el modelo BlogReaction existente
16. 🔴 Todo lo que funciona hoy sigue funcionando exactamente igual
17. 🟡 Tiempo de respuesta del endpoint < 80ms
18. 🟡 Código añadido < 3kb total
19. 🟡 Se reutiliza el middleware de rate limit existente
