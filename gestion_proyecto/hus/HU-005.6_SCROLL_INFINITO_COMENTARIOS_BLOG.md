# HU-005.6 - SCROLL INFINITO / CARGA INCREMENTAL AUTOMATICA COMENTARIOS BLOG

## ✅ Contexto
Cuando un blog tiene mas de 20 comentarios, la pagina se vuelve muy larga, el rendimiento baja y la experiencia de usuario es mala. Las redes sociales hace años solucionaron esto con carga incremental automatica.

---

## 🎯 Como usuario visitante
> Cuando leo un blog y hay muchos comentarios, quiero que se carguen automaticamente a medida que hago scroll, sin tener que clickear botones de paginacion, para una experiencia fluida y nativa.

---

## 📋 Criterios de Aceptación

### ✅ Funcionales
- [ ] Se cargan inicialmente **12 comentarios** (numero optimo UX)
- [ ] Cuando el usuario llega a **150px antes del ultimo comentario**, se cargan automaticamente los siguientes 12
- [ ] No hay botones, no hay recarga, no hay accion manual del usuario
- [ ] Se muestra un skeleton loader mientras se cargan los siguientes comentarios
- [ ] Cuando ya no hay mas comentarios, se muestra un mensaje final elegante
- [ ] Evitar cargas duplicadas mientras se esta haciendo la peticion
- [ ] Funciona con comentarios principales y respuestas anidadas
- [ ] Se mantienen todas las funcionalidades existentes: boton responder, reacciones, etc

### ✅ UX / Visual
- [ ] Animacion suave de aparicion de nuevos comentarios
- [ ] No hay saltos ni desplazamientos bruscos del scroll
- [ ] Skeleton loader igual al resto del sitio
- [ ] Mantener la paleta de colores azul/morado
- [ ] Mensaje final: `"✅ Has visto todos los comentarios"`

### ✅ Rendimiento
- [ ] Usar `IntersectionObserver API` (nativa del navegador, 10x mas rendimiento que escuchar evento scroll)
- [ ] Throttle automatico
- [ ] No hay perdida de performance incluso con 500+ comentarios
- [ ] Tiempo de respuesta < 200ms
