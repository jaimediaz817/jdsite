# 📋 HISTORIA DE USUARIO: HU-008
> Sistema de Avatares Consistentes para Comentarios del Blog sin Almacenamiento de Usuario
> Fecha: 29/04/2026
> Versión: 1.0
> Estado: 🔴 Planificado

---

## 🎯 Como visitante del blog
**Quiero** que cada comentario muestre un avatar con iniciales y color único que sea **siempre el mismo** para el mismo usuario
**Para** tener identidad visual consistente en mis comentarios sin tener que registrarme ni compartir datos personales

---

## 📊 Contexto técnico
- Sistema de comentarios YA implementado (`BlogComment` en `blog.models`)
- Sistema de reacciones a comentarios YA implementado (`CommentReaction` en `reactions.models`)
- Avatar debe ser determinístico: mismo usuario (IP + nombre) = avatar idéntico siempre
- Sin almacenar datos adicionales: solo propiedades calculadas
- GDPR-compliant: no persiste IP, solo calcula hash en tiempo real

---

## ✅ CRITERIOS DE ACEPTACIÓN

### Funcionales
1. 🔴 Avatar se genera basado en hash de (IP + nombre_comentarista)
2. 🔴 Avatar es **CONSISTENTE**: mismo usuario = avatar idéntico en múltiples visitas
3. 🔴 Avatar muestra iniciales del nombre (máx 2 caracteres)
4. 🔴 Color del avatar es único y consistente (derivado del hash)
5. 🔴 Avatar se muestra en comentarios principales Y respuestas anidadas
6. 🔴 Avatar se genera sin guardar ningún dato en BD (solo propiedades)
7. 🔴 Cambio de IP o nombre = avatar diferente (comportamiento esperado)
8. 🟡 Avatar visible incluso con Javascript desactivado

### Usabilidad
9. 🔴 Avatar tiene hover sutil (escala 1.05)
10. 🔴 Avatar integrado naturalmente con layout de comentarios
11. 🔴 Área mínima del avatar: 48x48px
12. 🔴 Tamaño de iniciales legible
13. 🟡 Avatar en móvil se ve igual que en desktop

### Técnicos
14. 🔴 NO se modifica modelo `BlogComment` (solo se agregan propiedades)
15. 🔴 NO se crea tabla nueva (propiedades calculadas, no campos BD)
16. 🔴 NO se instalan dependencias nuevas (solo hashlib nativo)
17. 🔴 Todo lo que funciona hoy sigue funcionando exactamente igual
18. 🔴 Propiedades `avatar_seed`, `avatar_color`, `avatar_initials` en BlogComment
19. 🟡 Código HTML/CSS/Lógica reutilizable para futuro sistema de usuario registrado

## 📝 CASOS DE PRUEBA OBLIGATORIOS
Todos los casos deben pasar:
- [ ] Usuario A comenta 2 veces = avatar idéntico ambas veces
- [ ] Usuario A desde otra IP = avatar diferente
- [ ] Usuario B comenta con otro nombre = avatar diferente
- [ ] Avatar visible en comentario principal
- [ ] Avatar visible en respuesta anidada
- [ ] Avatar visible sin JavaScript
- [ ] Recargar página = avatar igual
- [ ] Abrir en navegador diferente = avatar igual
- [ ] Color es hexadecimal válido
- [ ] Iniciales máximo 2 caracteres

## 🎨 ESPECIFICACIONES DE DISEÑO

### Avatar
- Forma: Círculo
- Tamaño: 48x48px
- Contenido: Iniciales (máx 2 caracteres)
- Color: Generado del hash (primera 6 chars)
- Fuente: Bold, 18px, color blanco
- Transición: transform 150ms ease

### Paletde colores
- Generada automáticamente del hash
- Garantiza contraste legible (blanco sobre color)
- Evita colores muy claros (agregar overlay si es necesario)

### Layout
```
[Avatar 48x48] Nombre
                 [Comentario multilinea]
                 [Reacciones]
                 [Responder]
```
