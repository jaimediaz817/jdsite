# HU-005.7 – Sidebar de filtrado por categoría y paginación mejorada

## Historia de Usuario
**Como** visitante del blog,
**quiero** disponer de un panel lateral que me permita filtrar los artículos por categoría y, cuando haya muchos artículos, navegar mediante una paginación clara,
**para** encontrar rápidamente el contenido de mi interés sin perderme en una lista interminable.

### Criterios de aceptación (UX/UI)
1. Sidebar visible en escritorio (≈250 px de ancho) con buen contraste.
2. En móvil, el sidebar se colapsa a un botón "Filtrar" que despliega la lista.
3. Enlaces accesibles: `aria-label`, foco visible y estado activo resaltado en púrpura (`#6f42c1`).
4. El filtro persiste al paginar (`?category=slug&page=n`).
5. Paginación centrada bajo el grid, con botones «Anterior», «Siguiente» y números limitados a ±2 del actual.
6. Feedback visual en hover (sombra ligera, cambio de color).
7. Carga de categorías única; paginación con `Paginator` (12 posts/página).

---

## Plan de implementación

### Backend
* **Vista (`backend/blog/views.py`)**: leer `category` del `request.GET`, filtrar el queryset, añadir `categories = Category.objects.all()` al contexto y usar `Paginator`.
* **URL (`backend/blog/urls.py`)**: mantener la ruta actual; los parámetros `category` y `page` se pasan como query string.

#### Detalles de implementación backend
```python
# backend/blog/views.py
from django.core.paginator import Paginator
from django.views.generic import ListView
from .models import Post, Category

class BlogListView(ListView):
    model = Post
    template_name = 'blog/blog_list.html'
    context_object_name = 'posts'
    paginate_by = 12  # número de posts por página

    def get_queryset(self):
        qs = super().get_queryset().filter(published=True).order_by('-publish_date')
        category_slug = self.request.GET.get('category')
        if category_slug:
            qs = qs.filter(category__slug=category_slug)
        return qs

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        # lista de todas las categorías para el sidebar
        context['categories'] = Category.objects.all()
        # conservar el filtro en los enlaces de paginación
        query = self.request.GET.copy()
        if 'page' in query:
            query.pop('page')
        context['query_string'] = query.urlencode()
        return context
```

En la plantilla `blog_list.html` los controles de paginación deben usar `{{ query_string }}` para mantener el filtro:
```html
{% if is_paginated %}
  <nav aria-label="Paginación">
    <ul class="pagination">
      {% if page_obj.has_previous %}
        <li class="page-item"><a class="page-link" href="?{{ query_string }}&page={{ page_obj.previous_page_number }}">Anterior</a></li>
      {% endif %}
      {% for num in page_obj.paginator.page_range %}
        {% if num == page_obj.number %}
          <li class="page-item active"><span class="page-link">{{ num }}</span></li>
        {% elif num > page_obj.number|add:'-3' and num < page_obj.number|add:'3' %}
          <li class="page-item"><a class="page-link" href="?{{ query_string }}&page={{ num }}">{{ num }}</a></li>
        {% endif %}
      {% endfor %}
      {% if page_obj.has_next %}
        <li class="page-item"><a class="page-link" href="?{{ query_string }}&page={{ page_obj.next_page_number }}">Siguiente</a></li>
      {% endif %}
    </ul>
  </nav>
{% endif %}
```

Esto garantiza que al cambiar de página el filtro de categoría se preserve y no haya errores en el frontend.
