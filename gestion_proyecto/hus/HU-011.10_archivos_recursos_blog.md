# HU-011.10 — Gestión de archivos/recursos asociados a artículos del blog

> **Origen:** El superadmin necesita una forma rápida de identificar qué artículos de prueba consumen más espacio en disco (imágenes, videos, etc.) para decidir cuáles eliminar completamente. Actualmente no hay visibilidad sobre los recursos físicos asociados a cada artículo desde la dashboard.

> **Objetivo:** Mostrar en la dashboard del superadmin la **cantidad de archivos estáticos** y el **tamaño total en disco** asociados a cada artículo, facilitando la toma de decisiones para limpieza de artículos de prueba y monitoreo de uso de recursos.

---

## 🟡 REGLAS DE ORO

1. **Aditivo:** No se modifica ningún comportamiento existente. Solo se añade información visual.
2. **Sin dependencias nuevas:** Se usa `os` y `pathlib` de la librería estándar de Python.
3. **Rendimiento:** El conteo de archivos se calcula bajo demanda (no se almacena en BD) ya que es una operación ligera en disco.
4. **Solo superadmin:** La información detallada de archivos es visible solo para `is_superuser=True`.
5. **Formato legible:** Los tamaños se muestran en formato humano (KB, MB, GB).
6. **Cero ruptura:** La tabla actual de la dashboard sigue funcionando igual, solo se añade una columna.

## 🔧 PLAN DE IMPLEMENTACIÓN (fases de ~10-15 min cada una)

### Fase 1: Función helper de conteo de archivos (~10 min)
- Crear función `get_post_files_info(post_slug)` en `backend/blog/services.py`:
  - Recibe el slug del artículo
  - Retorna diccionario: `{ 'file_count': int, 'total_size_bytes': int, 'total_size_human': str, 'files_by_type': dict }`
  - Usa `os.scandir()` para iterar archivos (más eficiente que `os.listdir()`)
  - Excluye `blog.md` del conteo (solo archivos multimedia/estáticos)
  - Formatea tamaño en KB/MB/GB
  - Retorna valores vacíos si la carpeta no existe

### Fase 2: Actualizar vista del dashboard (~10 min)
- Modificar `dashboard_view()` en `backend/blog/views.py`:
  - Para cada post en `page_obj`, agregar atributo `files_info` con el resultado de `get_post_files_info(post.slug)`
  - Solo calcular si el usuario es superadmin (para no afectar rendimiento de otros usuarios)
  - Agregar columna "📁 Archivos" al contexto

### Fase 3: Template de la dashboard (~15 min)
- Modificar `backend/blog/templates/blog/dashboard.html`:
  - Añadir columna "📁 Archivos" en el `<thead>` (solo visible para superadmin)
  - En cada fila, mostrar badge con cantidad de archivos y tamaño total
  - Ejemplo: `📁 5 archivos (2.3 MB)`
  - Si no hay archivos: mostrar `—`
  - Añadir tooltip o expandible con desglose por tipo (imágenes, videos, otros)
  - Estilo: badge similar al de comentarios pero en color naranja/ámbar

### Fase 4: Estilos CSS (~5 min)
- Añadir estilo para el badge de archivos en `dashboard.html` (dentro del `<style>` existente):
  - `.files-badge` con fondo ámbar/naranja claro
  - `.files-badge-high` para artículos con muchos archivos (>5) en rojo claro
  - Tooltip con desglose por tipo de archivo

### Fase 5: Prueba y validación (~10 min)
- Verificar que la columna aparece solo para superadmin
- Verificar que el conteo es correcto comparando con el sistema de archivos
- Verificar que el formato de tamaño es legible (KB, MB)
- Verificar que no afecta el rendimiento de la dashboard
- Verificar que no rompe ningún comportamiento existente
