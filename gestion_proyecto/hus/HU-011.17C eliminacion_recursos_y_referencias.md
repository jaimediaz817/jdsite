## Descripción general (contexto)

Cuando se elimina un recurso (imagen, video) desde el dashboard de recursos o desde el editor, **actualmente solo se elimina el archivo físico**. Sin embargo, ese recurso puede estar referenciado en múltiples lugares del contenido del artículo, lo que genera **errores 404** (imagen rota) en el blog público.

### Problema específico

Si elimino la imagen `image-1.png` físicamente pero no actualizo las referencias en el markdown, el artículo mostrará:
- Imágenes rotas en slides, galleries o popups
- Portada del blog sin cargar
- Cualquier referencia markdown que apunte a ese archivo

### Lugares donde se encuentran las referencias

| Ubicación                   | Ejemplo                                                                | Riesgo                    |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------- |
| **Frontmatter YAML**        | `image: "image-1.png"`                                                 | Portada del blog no carga |
| **Markdown inline**         | `![alt](image-1.png)`                                                  | Imagen rota en contenido  |
| **Bloque :::slides**        | `:::slides` + `![alt](image-1.png)`                                    | Slide roto                |
| **Bloque :::popup:gallery** | `:::popup:gallery` + `![alt](image-1.png)`                             | Gallery rota              |
| **HTML inline (blog.md)**   | `<img src="image-1.png" alt="...">`                                    | Imagen rota en contenido  |
| **Video inline (blog.md)**  | `<video src="video-01.mp4">` o `<source src="video-01.mp4">`           | Video roto en contenido   |
| **HTML compilado (BD)**     | `<img src="/static/blogs/slug/image-1.png">`                           | Imagen rota en público    |
| **Video compilado (BD)**    | `<video...><source src="/static/blogs/slug/video-01.mp4">`             | Video roto en público     |
| **Cualquier coincidencia**  | Cualquier referencia textual que contenga el nombre exacto del archivo | Error 404                 |

### Flujo actual problemático

```
Dashboard recursos → Eliminar imagen
  → Solo elimina archivo físico de static/blogs/<slug>/ y blogs_source/<slug>/
  → NO busca referencias en blog.md
  → NO actualiza HTML compilado en BD
  → Resultado: Error 404 en el blog público
```

## Objetivo

Implementar una **función de rastreo y actualización de referencias** que, al eliminar un recurso:
1. **Busque** todas las referencias al archivo en el contenido del artículo
2. **Muestre** al usuario dónde está referenciado el archivo
3. **Pregunte** si desea eliminar también las referencias
4. **Actualice** el markdown (blog.md) eliminando/comentando las referencias
5. **Recompile** el HTML actualizado en la base de datos

## 🔍 Diagnóstico detallado

### Escenario 1: Eliminar imagen desde Dashboard de Recursos

**Flujo actual:**
```
1. Admin selecciona imagen en dashboard de recursos
2. Hace clic en "Eliminar"
3. Backend elimina archivo de:
   - backend/static/blogs/<slug>/image-1.png
   - backend/blogs_source/<slug>/image-1.png
4. NO se actualiza blog.md
5. NO se recompila content_html en BD
6. Blog público muestra: <img src="/static/blogs/slug/image-1.png"> → 404
```

**Flujo deseado:**
```
1. Admin selecciona imagen en dashboard de recursos
2. Hace clic en "Eliminar"
3. Backend busca referencias en blog.md:
   - Frontmatter: image: "image-1.png"
   - Markdown: ![alt](image-1.png)
   - Bloques: :::slides, :::popup:gallery
4. Muestra al usuario: "Esta imagen está referenciada en:
   - Portada del artículo (frontmatter)
   - Línea 45 del markdown (slides)
   - Línea 78 del markdown (gallery)"
5. Pregunta: "¿Desea eliminar también las referencias?"
6. Si SÍ:
   - Actualiza blog.md eliminando/comentando las referencias
   - Re-importa el artículo (import_blogs)
   - Recompila content_html en BD
7. Elimina el archivo físico
8. Blog público: sin errores 404
```

### Escenario 2: Eliminar imagen desde Editor (blog_editor.js)

**Flujo actual:**
```
1. Editor muestra lista de archivos subidos
2. Usuario hace clic en icono de eliminar (🗑️)
3. Se ejecuta removeUploadedFile(filename):
   - Elimina de uploadedFiles array
   - Llama removeMarkdownLineForFile(filename)
   - Elimina vista previa del DOM
   - Llama deleteFileOnServer(filename)
4. removeMarkdownLineForFile solo elimina líneas simples de markdown
5. NO maneja bloques especiales (:::slides, :::popup:gallery)
6. Al guardar, se re-importa pero las referencias rotas persisten
```

**Flujo deseado:**
```
1. Editor muestra lista de archivos subidos
2. Usuario hace clic en icono de eliminar (🗑️)
3. Se ejecuta removeUploadedFile(filename):
   - Busca TODAS las referencias (frontmatter, markdown, bloques especiales)
   - Muestra al usuario dónde está referenciado
   - Pregunta confirmación
   - Elimina todas las referencias
   - Elimina archivo físico
4. Al guardar, el markdown está limpio
5. Blog público: sin errores 404
```

### Escenario 3: Eliminar imagen desde línea de comandos (script)

**Flujo actual:**
```
1. Admin ejecuta script para limpiar imágenes huérfanas
2. Script elimina archivos sin verificar referencias
3. Artículos quedan con imágenes rotas
```

**Flujo deseado:**
```
1. Admin ejecuta script para limpiar imágenes huérfanas
2. Script primero verifica si la imagen está referenciada en algún blog.md
3. Si está referenciada, la omite o pregunta
4. Solo elimina imágenes realmente huérfanas
```

### Patrones de búsqueda

```python
# 1. Frontmatter YAML
frontmatter_patterns = [
    r'^image:\s*["\']?([^"\']*\{filename\}[^"\']*)["\']?',  # image: "file.png"
    r'^cover_image:\s*["\']?([^"\']*\{filename\}[^"\']*)["\']?',  # cover_image: "file.png"
    r'^images:',  # Array de imágenes (requiere parseo YAML)
]

# 2. Markdown inline
markdown_patterns = [
    r'!\[[^\]]*\]\([^)]*\{filename\}[^)]*\)',  # ![alt](file.png)
]

# 3. HTML inline (img, video, source)
html_patterns = [
    r'<img\s+[^>]*src=["\'][^"\']*{filename}[^"\']*["\']',  # <img src="file.png">
    r'<img\s+[^>]*src=["\'][^"\']*{filename}[^"\']*["\']',  # <img src='file.png'>
    r'<video\s+[^>]*src=["\'][^"\']*{filename}[^"\']*["\']',  # <video src="file.mp4">
    r'<source\s+[^>]*src=["\'][^"\']*{filename}[^"\']*["\']',  # <source src="file.mp4">
]

# 4. Bloques especiales (entre ::: y :::)
block_patterns = {
    'slides': r':::slides\n(.*?)\n:::',
    'gallery': r':::popup:gallery\n(.*?)\n:::',
    'callout': r':::callout:\w+\n(.*?)\n:::',
}

# 5. Coincidencia exacta (fallback) - busca el nombre exacto del archivo
# Se usa como último recurso para detectar cualquier referencia
exact_match_pattern = re.compile(r'(?<![/\w]){re.escape(filename)}(?![/\w])')
# Nota: usa lookbehind y lookahead para evitar falsos positivos
# Ejemplo: "image-1.png" no coincidiría con "my-image-1.png" pero sí con "image-1.png"
```

### Algoritmo de eliminación

```python
def remove_references_from_markdown(blog_md_path: Path, filename: str, references: list) -> str:
    """
    Elimina las referencias a un archivo del blog.md.
    
    Returns:
        Contenido actualizado del blog.md
    """
    lines = blog_md_path.read_text(encoding='utf-8').split('\n')
    
    # Ordenar referencias por línea descendente para no afectar índices
    sorted_refs = sorted(references, key=lambda x: x['line'], reverse=True)
    
    for ref in sorted_refs:
        line_idx = ref['line'] - 1  # Convertir a 0-based
        
        if ref['type'] == 'frontmatter':
            # Eliminar la línea completa del frontmatter
            lines.pop(line_idx)
        
        elif ref['type'] == 'markdown':
            # Eliminar solo la línea del markdown
            lines.pop(line_idx)
        
        elif ref['type'] in ('slides', 'gallery', 'callout'):
            # Verificar si es la única imagen del bloque
            # Si es así, eliminar todo el bloque
            # Si hay más imágenes, solo eliminar la línea
            block_start = find_block_start(lines, line_idx)
            block_end = find_block_end(lines, line_idx)
            
            images_in_block = count_images_in_block(lines, block_start, block_end)
            
            if images_in_block <= 1:
                # Eliminar todo el bloque
                del lines[block_start:block_end + 1]
            else:
                # Solo eliminar la línea
                lines.pop(line_idx)
    
    return '\n'.join(lines)
```
