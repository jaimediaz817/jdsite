# HU-011.18: Rediseño del Footer del Blog con criterios UX/UI profesionales

## Objetivo
Transformar el footer del blog para que tenga una estética moderna, profesional y responsive, reemplazando el diseño actual de apariencia "universitaria/gobierno" por una experiencia visual acorde a un blog personal de calidad.

---

## Criterios de aceptación

### 1. Estructura visual moderna
- Jerarquía visual clara con tipografía diferenciada (títulos vs. enlaces vs. metadata)
- Uso estratégico de espacios en blanco (whitespace)
- Separación visual entre secciones mediante divisores sutiles o espaciado
- Alineación precisa y consistente (grid system)

### 2. Paleta de colores profesional
- Fondo oscuro con contraste suficiente para legibilidad (WCAG AA mínimo)
- Colores de acento coherentes con la identidad del blog
- Estados hover/focus diferenciados para interacciones
- Evitar colores saturados o combinaciones estridentes

### 3. Tipografía jerárquica
- Títulos de sección con peso y tamaño diferenciado
- Enlaces con legibilidad óptima (tamaño mínimo 14px)
- Información secundaria (copyright) con jerarquía inferior
- Line-height y letter-spacing optimizados para lectura

### 4. Responsive Design (Mobile First)
- **Móvil (< 768px):** Una columna apilada verticalmente
- **Tablet (768px - 1024px):** 2 columnas o rejilla adaptativa
- **Desktop (> 1024px):** 4 columnas máximo o diseño expandido
- Touch targets mínimo 44x44px para enlaces en móvil
- Espaciado adaptativo según viewport

### 5. Componentes mejorados
- **Logo:** Mejor integración con el footer (tamaño proporcional, alignment)
- **Navegación:** Organización lógica por grupos relacionados
- **Posts recientes:** Formato más scannable (fechas opcionales, truncamiento inteligente)
- **Categorías/Etiquetas:** Presentación visual más atractiva (badges o pills opcional)
- **Redes sociales:** Iconografía clara con etiquetas descriptivas
- **Newsletter (opcional):** Espacio reservado para crecimiento futuro

### 6. Microinteracciones sutiles
- Transiciones suaves en hover (color, transform, opacity)
- Feedback visual en enlaces (subrayado animado, desplazamiento)
- Efectos de profundidad sutiles (sombras, elevación)
- **Sin exageraciones:** EFECTOS RESPONSABLES que mejoren UX, no la distraigan

### 7. Accesibilidad
- Contraste mínimo 4.5:1 para texto normal, 3:1 para texto grande
- Focus states visibles para navegación por teclado
- Textos alternativos descriptivos en iconos/logo
- Semántica HTML correcta (`<nav>`, `<address>`, etc.)

### 8. Rendimiento
- CSS optimizado (sin recursos innecesarios)
- Iconos vectoriales (Font Awesome actual o SVG)
- Sin dependencias de JavaScript para funcionalidad básica
- Critical CSS inline si es necesario (pero no crítico)

## Notas de diseño

**Inspiración estética (referencia):** Sitios como Medium, Ghost,Svbtle, o blogs de diseñadores reconocidos. Estilo limpio, tipográfico, con buen uso del espacio.

**Anti-patrones a evitar:**
- ❌ Footer "acordeón" en móvil (mejor scroll natural)
- ❌ Colores neón o saturados
- ❌ Tipografía decorativa ilegible
- ❌ Animaciones excesivas (parallax, bouncing, etc.)
- ❌ Información irrelevante o redundante
- ❌ Diseño genérico "template Bootstrap 3 sin customizar"

**Referencias de UX:**
- Ley de Fitts: botones/enlaces accesibles
 - Ley de Hick: limitar opciones (max 7 enlaces por sección)
- Principio de proximidad: agrupar items relacionados
- Principio de contraste: jerarquía visual clara
