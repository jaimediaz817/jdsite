# HU-011.2: Sincronización de formato Markdown con bloques de portada y galería

## Contexto
Permitir que, al crear un artículo desde el editor, se genere automáticamente un bloque Markdown que defina la imagen de portada y la galería de imágenes según el formato especificado (ej.: `:::cover`, `:::gallery`). Esto garantiza que las imágenes se rendericen con las reglas de formato mínimas descritas en la documentación.

## Objetivo
- Definir una sintaxis clara y consistente para bloques de portada y galería.
- Asegurar que el editor genere el bloque correcto al arrastrar y soltar imágenes.
- Mantener la compatibilidad con los procesadores Markdown actuales.

## Criterios de aceptación
1. Al arrastrar una imagen al editor, se crea automáticamente un bloque `:::cover` con la ruta de la imagen.
2. Al arrastrar varias imágenes, se crea un bloque `:::gallery` que contiene cada imagen dentro de su propia etiqueta.
3. Los bloques generados siguen el formato mínimo requerido (sin estilos adicionales).
4. El procesador Markdown reconoce y renderiza correctamente los bloques `:::cover` y `:::gallery`.
5. La documentación del usuario se actualiza con ejemplos de uso.

## Tareas
- [ ] Analizar el procesador Markdown actual (`markdown_utils.py`).
- [ ] Proponer sintaxis para `:::cover` y `:::gallery`.
- [ ] Implementar la generación de bloques en el editor.
- [ ] Actualizar `markdown_utils.py` para parsear los nuevos bloques.
- [ ] Testear que la salida sea correcta.
- [ ] Documentar el nuevo formato en la guía de usuario.

## Dependencias
- Ninguna nueva dependencia externa.
- Modificaciones en `backend/blog/utils/importer/markdown_utils.py`.
- Cambios menores en `backend/blog/static/blog/js/blog_editor.js` (opcional).
