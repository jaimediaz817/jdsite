# HU-011.5 — Dashboard de Aprobación de Borradores

> **Origen:** Al guardar un artículo desde el editor, los usuarios sin privilegios lo dejan en `draft: true` y no existe un proceso claro para que el admin lo apruebe. Los borradores se acumulan invisibles en la base de datos.  
> **Objetivo:** Que cualquier artículo creado o editado por un usuario no‑admin pase por un flujo de moderación: email al admin → dashboard `/dashboard/` → aprobar o rechazar con comentario → email automático al autor con el resultado.

---

## 🟡 REGLAS DE ORO

1. **Admin = publica directo** (sin pasar por moderación). Su botón sigue siendo “🟢 Publicar”.  
2. **No‑admin = siempre borrador** (aunque marque “Publicar inmediatamente”). El admin debe validar.  
3. **Email automático al admin** cuando se crea un nuevo borrador.  
4. **Email automático al autor** con el resultado (aprobado/rechazado + comentario).  
5. **Dashboard accesible solo para superusuarios** (staff o superuser).
