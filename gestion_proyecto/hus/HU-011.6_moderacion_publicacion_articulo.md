# HU-011.6 — Moderación y publicación vía URL de aprobación

> **Origen:** Al guardar un artículo desde el editor como borrador, el sistema muestra el mensaje *“Borrador guardado. Pendiente de aprobación.”* y envía un email al administrador. Actualmente el admin debe entrar al panel de Django para cambiar el estado del `BlogPost`. Queremos simplificar este proceso con un enlace de aprobación que el admin pueda activar directamente desde el email.

> **Objetivo:** Permitir que, cuando un usuario **no‑admin** guarda un borrador, el administrador reciba un email con una **URL única de aprobación**. Al hacer clic en esa URL el artículo se publica automáticamente (cambia `is_published` a `True`, `moderation_status` a `approved`, se crea un registro en `BlogModeration`) y se notifica al autor.

---

## 🟡 REGLAS DE ORO

1. **Envío de email al admin** al crear o actualizar un borrador (`draft: true`).  
2. **URL de aprobación única** (`/approve/<token>/`) que solo puede usarse una vez y expira a los **48 h**.  
3. **Acción automática** al acceder a la URL:
   - Marca el `BlogPost` como publicado (`is_published=True`).
   - Cambia `moderation_status` a `approved`.
   - Crea/actualiza el registro `BlogModeration` con `action='approved'`.
   - Envía email al autor con el resultado.
4. **Seguridad:** la URL contiene un token aleatorio asociado al `BlogPost` y solo es válida para usuarios con `is_staff` o `is_superuser`.  
5. **Idempotencia:** si la URL se usa después de haber sido consumida o expiró, se muestra un mensaje de error y no se modifica nada.

## 🔧 PASOS DE IMPLEMENTACIÓN (≈15 min cada uno)

1. **Modelo `BlogPost`** – añadir campos `approval_token` (CharField, nullable) y `approval_token_created` (DateTimeField).  
2. **Generación del token** – en `save_blog_to_source` (o en `BlogPost.save`) generar un UUID cuando `draft=True` y no exista token; guardar timestamp.  
3. **Vista de aprobación** – crear `approve_blog_view(request, token)`:
   - Buscar `BlogPost` por `approval_token`.
   - Verificar que el token no haya expirado y que el usuario sea staff.
   - Actualizar `is_published=True`, `moderation_status='approved'`.
   - Crear/actualizar `BlogModeration` con `action='approved'`.
   - Invalidar el token (`approval_token=None`).
   - Enviar email al autor (plantilla `email_approval_success.html`).
   - Redirigir a una página de confirmación.
4. **URL** – añadir en `backend/blog/urls.py`:
   ```python
   path('approve/<slug:token>/', views.approve_blog_view, name='approve_blog')
   ```
5. **Email al admin** – actualizar la función que envía el email de notificación para incluir la URL:
   ```
   https://{{ site_domain }}/approve/{{ post.approval_token }}/
   ```
