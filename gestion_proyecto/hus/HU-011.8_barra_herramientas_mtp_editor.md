# HU-011.8: Barra de herramientas MTP (Mark to Post) en el editor

**Estado:** ✅ Pendiente  
**Prioridad:** Alta  
**Dependencias:** HU-011, HU-017  

---

## 🎯 Objetivo

Agregar una **barra de herramientas flotante en la derecha del editor** que permita insertar bloques de sintaxis MTP (Mark to Post) con un solo clic, tipo botones estilo MS Word, para que escribir en el formato MTP sea rápido, visual y sin fricción.

---

## 🧠 Contexto

Actualmente el escritor debe recordar y escribir manualmente la sintaxis de cada bloque MTP:

```
:::slides
![alt](image.png)
:::

:::callout:info
Texto aquí
:::
```

Con la barra MTP, un clic inserta el template completo en el editor. El usuario solo rellena contenido.

---

## ✅ Criterios de Aceptación

1. **Barra flotante visible a la derecha del editor** en posición fixed, no estorba el contenido
2. **Botones con iconos claros y tooltips** (FontAwesome ya cargado en el proyecto)
3. **Inserción inteligente**: cada botón inserta un template MTP completo en la posición del cursor
4. **Cobertura de todos los bloques MTP**:
   - [ ] Imagen de portada (frontmatter `image:`)
   - [ ] `:::slides` → template slides
   - [ ] `:::callout:info` → template callout info
   - [ ] `:::callout:warning` → template callout warning
   - [ ] `:::callout:tip` → template callout tip
   - [ ] `:::pullquote` → template pullquote
   - [ ] `:::codefile:ruta` → template codefile
   - [ ] `:::popup:gallery` → template popup gallery
   - [ ] `[vl]: highlight` → template visual layout
   - [ ] Separador `---`
   - [ ] Video `<video>` template
   - [ ] Imagen markdown estándar
5. **Responsive**: en mobile la barra se colapsa o se vuelve horizontal abajo
6. **Sin dependencias nuevas**: solo CSS + JS vanilla (lo que ya existe)

## 📦 Fases de Implementación

### Fase 1: CSS de la barra MTP
- Posición fixed derecha
- Fondo semitransparente, bordes redondeados
- Botones cuadrados con iconos grandes
- Tooltips al hover

### Fase 2: HTML de la barra en el editor
- Contenedor `<aside class="mtp-toolbar">` dentro del `.editor-wrapper`
- Botones con `data-mtp-action="slides"` etc.

### Fase 3: JS - motor de inserción
- Función central `insertMtpTemplate(action)` que usa `easyMDE.codemirror.replaceSelection()`
- Cada acción mapea a un template MTP

### Fase 4: Prueba
- Abrir editor, hacer clic en cada botón, verificar inserción correcta
- Verificar responsive
- Verificar que no rompe el autosave ni otras funcionalidades
