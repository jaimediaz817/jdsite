# HU-011.9 — Eliminación permanente de artículos (superadmin)

> **Origen:** Actualmente el superadmin solo puede activar/desactivar artículos desde la dashboard, pero no puede eliminarlos permanentemente. Cuando un artículo es inapropiado, spam o debe ser removido del sistema completamente, no existe ninguna forma de hacerlo sin acceder a la consola de Django o a la base de datos directamente.

> **Objetivo:** Implementar un botón de **eliminación permanente** en la dashboard del superadmin que borre completamente un artículo de todo rastro: base de datos (BlogPost + registros asociados BlogModeration) y recursos físicos (carpeta del artículo con todas sus imágenes y archivos multimedia).

---

## 🟡 REGLAS DE ORO

1. **Solo superadmin:** La función de eliminación permanente solo está disponible para `is_superuser=True`.
2. **Confirmación obligatoria:** Antes de eliminar, se muestra un diálogo de confirmación que advierte que la acción es irreversible.
3. **Eliminación completa:** Se borra el registro de `BlogPost`, todos los `BlogModeration` asociados, y la carpeta física del artículo en `blogs_source/<slug>/`.
4. **Log de auditoría:** Se registra qué usuario eliminó qué artículo y cuándo (en el mensaje de Django messages).
5. **Redirección:** Después de eliminar, se redirige a la dashboard con un mensaje de éxito/error.
6. **Cero ruptura:** No se modifica ningún comportamiento existente. Solo se añade una nueva funcionalidad aditiva.

---

## 📋 FLUJO DE ELIMINACIÓN

1. El superadmin ve la dashboard y localiza el botón 🗑️ (papelera) junto a cada artículo.
2. Al hacer clic, aparece un modal de confirmación: "¿Eliminar permanentemente '[Título]'? Esta acción no se puede deshacer."
3. Si confirma, se ejecuta:
   - `BlogModeration.objects.filter(blog_post=post).delete()`
   - `shutil.rmtree(blogs_source/<folder>/)` (elimina todos los archivos del artículo)
   - `post.delete()` (elimina el registro de la base de datos)
4. Se muestra un mensaje de éxito: "Artículo '[Título]' eliminado permanentemente."

---

## 🔧 PLAN DE IMPLEMENTACIÓN

### Fase 1: Modelo y lógica de eliminación (~10 min)
- Crear función `delete_post_permanently(post)` en `backend/blog/services.py`:
  1. Eliminar `BlogModeration` asociados al post
  2. Eliminar carpeta física del artículo en `blogs_source/<folder>/`
  3. Eliminar el `BlogPost` de la base de datos
  4. Registrar en logs la acción
- Crear vista `delete_blog_view` en `views.py`:
  - Solo superadmin
  - Verificar que el post exista
  - Llamar a `delete_post_permanently(post)`
  - Mostrar mensaje de éxito con Django messages
  - Redirigir a la dashboard

### Fase 2: URL + Template (~10 min)
- Añadir URL: `path('delete/<int:post_id>/', views.delete_blog_view, name='delete_blog')`
- Modificar `dashboard.html` para añadir botón de papelera junto a cada artículo
- Añadir JavaScript para mostrar modal de confirmación antes de eliminar
- Estilo del botón: rojo con ícono `fa-trash`
