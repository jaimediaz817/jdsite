# HU-018.1: Newsletter desde el Footer + Sistema de Plantillas de Email Reutilizables v1

> **Origen:** Durante el rediseño del footer profesional (HU-011.18), se agregó un input de captura de email en la columna Brand. Sin embargo, este input es un placeholder funcional sin propuesta de valor, sin beneficio claro para el usuario, y sin un backend que procese realmente la suscripción. Adicionalmente, el blog ya envía emails (comentarios, borradores, ver HU-011.85) pero sin una plantilla HTML estandarizada — van en texto plano.

**Objetivo dual:**
1. Convertir el input del footer en un **Newsletter funcional** con propuesta de valor clara
2. Crear un **sistema de plantillas de email reutilizables** con templates HTML estandarizados (base + variantes) para TODO envío del blog

---

## 🟡 REGLAS DE ORO

1. **No romper nada existente:** El footer actual y los envíos de email actuales siguen funcionando
2. **Tabla NewsletterSubscriber** extensible para segmentación futura
3. **Templates de email en HTML real** (no texto plano), con versión texto plano como fallback
4. **Base template de email** (header + footer + estilos inline) reusable por todos los flujos
5. **Sin nuevas dependencias:** Usar `django.core.mail` + `EmailMultiAlternatives`
6. **El newsletter es opt-in explícito:** Doble confirmación (double opt-in) para evitar spam y cumplir GDPR
7. **Valor inmediato al suscribirse:** No pedir el email sin decir qué obtendrán

---

## 🧠 DIAGNÓSTICO: El problema del newsletter actual

| Aspecto                | Estado actual | Problema                                                |

| **Propuesta de valor** | ❌ Ausente     | "Tu mejor email" no dice nada. ¿Por qué daría mi email? |

## 🎯 PROPUESTA DE VALOR DIFERENCIAL (ideas)

### Lo que recibirá el suscriptor (el "para qué"):

```
✅ 1 email por semana (máximo) — sin spam
✅ Resumen del artículo más importante de la semana
✅ Tips exclusivos de productividad para devs (que NO publico en el blog)
✅ Acceso anticipado a borradores antes de publicación pública
✅ Ocasionalmente: mini-guías PDF descargables (ej: "Checklist para lanzar un blog técnico")
```

### Diferenciador clave — "El Autor Responde":

Esta es la idea que **realmente diferencia** el newsletter de cualquier otro:

> **Cada suscriptor puede responder al email y el autor le responde personalmente.**
> No es un "no-reply@". Cada newsletter se envía con `Reply-To` configurado.
> El autor se compromete a responder personalmente cada respuesta en máximo 48 hrs.

Esto convierte el newsletter en **un canal de conversación directa** con la audiencia, no en un broadcast unidireccional. Es un diferenciador brutal vs newsletters masivas.

### Props de confianza visibles en el footer:

Junto al input, mostrar micro-indicadores:
```
📬 1 email/semana · Sin spam · Respondo personalmente
```

### Fase 1: Modelo NewsletterSubscriber (~10 min)

```python
class NewsletterSubscriber(models.Model):
    email = models.EmailField(unique=True)
    is_active = models.BooleanField(default=False)  # Se activa tras confirmar
    token = models.CharField(max_length=64, unique=True)  # Para confirmación
    subscribed_at = models.DateTimeField(auto_now_add=True)
    confirmed_at = models.DateTimeField(null=True, blank=True)
    unsubscribed_at = models.DateTimeField(null=True, blank=True)
    metadata = models.JSONField(default=dict, blank=True)  # User-Agent, fuente, etc.
    
    class Meta:
        verbose_name = "Suscriptor Newsletter"
        verbose_name_plural = "Suscriptores Newsletter"
        ordering = ['-subscribed_at']
```

### Fase 2: Backend de suscripción (~15 min)

- **Vista `newsletter_subscribe`** (POST): Recibe email, crea `NewsletterSubscriber`, envía email de confirmación (con token), retorna JSON
- **Vista `newsletter_confirm`** (GET): Confirma con token, activa `is_active=True`
- **Vista `newsletter_unsubscribe`** (GET): Desuscribe con token
- **URLs en `blog/urls.py`**:
  ```python
  path('newsletter/subscribe/', views.newsletter_subscribe, name='newsletter_subscribe'),
  path('newsletter/confirm/<slug:token>/', views.newsletter_confirm, name='newsletter_confirm'),
  path('newsletter/unsubscribe/<slug:token>/', views.newsletter_unsubscribe, name='newsletter_unsubscribe'),
  ```

### Fase 4: Helper para enviar emails con template (~10 min)

Crear en `services.py`:
```python
def send_templated_email(subject, template_name, context, to_email, from_email=None):
    """
    Envía un email usando template HTML + texto plano.
    template_name: ruta relativa a templates/email/ (sin extensión)
    context: dict con variables del template
    """
    html_template = get_template(f'email/{template_name}.html')
    txt_template = get_template(f'email/{template_name}.txt')
    
    html_content = html_template.render(context)
    txt_content = txt_template.render(context)
    
    msg = EmailMultiAlternatives(
        subject=subject,
        body=txt_content,
        from_email=from_email or settings.DEFAULT_FROM_EMAIL,
        to=[to_email],
        reply_to=[settings.OWNER_EMAIL],  # ✅ Diferenciador: se puede responder
    )
    msg.attach_alternative(html_content, "text/html")
    return msg.send()
```

Migrar los `send_mail()` existentes (comentarios, borradores) a usar `send_templated_email()`.

### Fase 6: JS de suscripción AJAX (~10 min)

```javascript
// newsletter.js — Maneja la suscripción vía fetch
document.addEventListener('DOMContentLoaded', function() {
    const form = document.getElementById('newsletter-form');
    if (!form) return;
    
    form.addEventListener('submit', async function(e) {
        e.preventDefault();
        const input = this.querySelector('.newsletter-input');
        const feedback = document.getElementById('newsletter-feedback');
        const email = input.value.trim();
        
        if (!email || !email.includes('@')) {
            showFeedback('Por favor ingresa un email válido', 'error');
            return;
        }
        
        try {
            const response = await fetch('/blog/newsletter/subscribe/', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/x-www-form-urlencoded',
                    'X-CSRFToken': getCSRFToken(),
                },
                body: `email=${encodeURIComponent(email)}`
            });
            
            const data = await response.json();
            
            if (response.ok) {
                showFeedback('✅ Revisa tu correo para confirmar la suscripción', 'success');
                input.value = '';
            } else {
                showFeedback(data.error || 'Error al suscribirte. Intenta de nuevo.', 'error');
            }
        } catch (err) {
            showFeedback('Error de conexión. Intenta más tarde.', 'error');
        }
    });
    
    function showFeedback(msg, type) {
        const el = document.getElementById('newsletter-feedback');
        el.textContent = msg;
        el.className = 'newsletter-feedback ' + type;
        el.style.display = 'block';
        setTimeout(() => { el.style.display = 'none'; }, 6000);
    }
    
    function getCSRFToken() {
        return document.querySelector('[name=csrfmiddlewaretoken]')?.value || '';
    }
});
```

        │
        ▼

        │
        ▼
  Respuesta: "✅ Revisa tu correo para confirmar"
        │
        ▼
  Usuario hace click en enlace de confirmación
        │
        ▼
  [GET /blog/newsletter/confirm/<token>/]
        │
        ├─ Token válido ──→ is_active=True, confirmed_at=now
        │                     └─→ Enviar email bienvenida (template: newsletter/welcome.html)
        │
        └─ Token inválido/expirado ──→ Error: "Enlace inválido"
