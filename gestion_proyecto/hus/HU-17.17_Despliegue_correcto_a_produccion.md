# HU-17.17 Despliegue correcto a producción

## Objetivo
Garantizar que el proyecto se despliegue en el entorno de producción de forma fiable, incluyendo la actualización de dependencias, compilación de assets estáticos, generación de migraciones y reinicio del servidor.

## Criterios de aceptación
- `requirements.txt` contiene todas las dependencias, incluyendo `black`, `ruff` e `isort`.
- Los archivos estáticos de Django están compilados (`collectstatic`).
- Todas las migraciones pendientes se han creado y aplicado.
- Los archivos SASS se han compilado a CSS.
- El código está versionado y el repositorio en la VPS está actualizado.
- El servicio Gunicorn se ha reiniciado sin errores y la aplicación responde correctamente.

## Pasos de implementación
1. **Actualizar `requirements.txt**  
   - Instalar paquetes nuevos si no están presentes: `black`, `ruff`, `isort`.  
   - Ejecutar `pip freeze > requirements.txt`.
2. **Compilar archivos estáticos**  
   ```bash
   cd backend
   python manage.py collectstatic --noinput
   ```
3. **Crear y aplicar migraciones**  
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```
4. **Compilar SASS**  
   ```bash
   npm run sass:watch   # o npm run sass para compilación única
   ```
5. **Actualizar repositorio en la VPS**  
   - Local: `git push origin main`  
   - VPS:  
     ```bash
     cd /var/www/jdiaz.tipsterbyte.com/app
     git fetch --all && git reset --hard origin/main
     source env/bin/activate
     pip install -r backend/requirements.txt
     ```
6. **Reiniciar el servicio Gunicorn**  
   ```bash
   sudo systemctl restart jdiaz_gunicorn.service
   sudo systemctl status jdiaz_gunicorn.service --no-pager
   ```
7. **Verificar despliegue**  
   Acceder a la URL de producción y comprobar que la aplicación funciona sin errores.

## Notas adicionales
- Asegurarse de que el archivo `.env` en la VPS contiene todas las variables de entorno necesarias.  
- Si se añaden nuevas dependencias de Python, repetir el paso 1.  
- Ejecutar `black .`, `isort .` y `ruff check . --fix` antes de generar `requirements.txt` para mantener el código limpio.
