> **Propósito:** Este documento es un procedimiento operativo estándar (POE).

## 📋 Prerrequisitos

Antes de iniciar el despliegue, confirmar que se cumplen estas condiciones:

| #   | Requisito                                             | Cómo verificarlo                                                    |
| --- | ----------------------------------------------------- | ------------------------------------------------------------------- |
| 1   | Tener acceso SSH a la VPS                             | `ssh usuario@vps-ip` debe conectar sin errores                      |
| 2   | Tener permisos `sudo` en la VPS                       | `sudo -v` debe pedir contraseña (si no expiró)                      |
| 3   | Repositorio local en `main` sin commits sin push      | `git log --oneline origin/main..HEAD` debe estar vacío              |
| 4   | `.env` en la VPS tiene todas las variables necesarias | `cat /var/www/jdiaz.tipsterbyte.com/app/.env` — revisar manualmente |
| 5   | Puerto de la app accesible (80/443)                   | `curl -I https://jaimediaz.dev` debe responder `200`                |
| 6   | Servicio Gunicorn existe y está habilitado            | `sudo systemctl is-enabled jdiaz_gunicorn.service` → `enabled`      |

## 🪜 Procedimiento paso a paso

> ⚠️ **Importante:** Cada paso tiene una **verificación** explícita.  
> No avanzar al siguiente paso si la verificación falla.  
> Si falla, ir a la sección [Solución de problemas](#-solucion-de-problemas).

---

### FASE 1 — Preparación local (máquina de desarrollo)

#### Paso 1.1 — Lint y formateo del código

```bash
# Desde la raíz del proyecto
source .venv/Scripts/activate
cd backend

black .
isort .
ruff check . --fix
```

**Verificación:** Los comandos deben terminar con código 0.  
Si `ruff` reporta errores que no puede auto-fix, revisarlos manualmente con `ruff check .`.

#### Paso 1.2 — Congelar dependencias

```bash
pip freeze > ../requirements.txt
```

**Verificación:** 
```bash
grep -E "^(black|ruff|isort)==" ../requirements.txt
```
Debe mostrar las 3 líneas con sus versiones.

#### Paso 1.3 — Compilar SASS a CSS (si hay cambios en archivos `.scss`)

```bash
cd ..
npm run sass
```

**Verificación:** 
```bash
git diff --name-only -- '*.css'
```
Si hay archivos `.css` modificados, significa que SASS generó cambios. Verificar que se vean correctos.

#### Paso 1.4 — Commit y push

```bash
git add -A
git commit -m "deploy: preparacion para despliegue $(date +%Y-%m-%d_%H%M)"
git push origin main
```

**Verificación:**
```bash
git log --oneline origin/main..HEAD
```
Debe estar vacío (no hay commits locales sin pushear).

---

### FASE 2 — Despliegue en VPS

#### Paso 2.1 — Conectar a la VPS

```bash
ssh usuario@vps-ip
```

#### Paso 2.2 — Actualizar el repositorio

```bash
cd /var/www/jdiaz.tipsterbyte.com/app
git fetch --all && git reset --hard origin/main
```

**Verificación:**
```bash
git log --oneline -3
```
Debe mostrar los últimos commits locales. El primer commit debe ser el mismo que acabas de pushear.

#### Paso 2.3 — Activar entorno virtual e instalar dependencias

```bash
source env/bin/activate
pip install -r backend/requirements.txt
```

**Verificación:**
```bash
pip list | grep -E "^(black|ruff|isort)"
```
Debe mostrar las 3 herramientas con sus versiones instaladas.

#### Paso 2.4 — Aplicar migraciones de base de datos

```bash
cd backend
python manage.py migrate
```

**Verificación:**
```bash
python manage.py showmigrations | grep "\[ \]"
```
No debe mostrar ninguna migración sin aplicar.

#### Paso 2.5 — Compilar archivos estáticos

```bash
python manage.py collectstatic --noinput
```

**Verificación:**
```bash
echo $?
```
Debe ser `0`.

---

### FASE 3 — Post-despliegue

#### Paso 3.1 — Reiniciar Gunicorn

```bash
sudo systemctl restart jdiaz_gunicorn.service
sudo systemctl status jdiaz_gunicorn.service --no-pager
```

**Verificación:** El estado debe mostrar:
- `● jdiaz_gunicorn.service - Gunicorn`
- `Active: active (running)` en **verde**
- Sin errores en las últimas líneas de log

#### Paso 3.2 — Verificar health-check de la aplicación

```bash
curl -s -o /dev/null -w "HTTP %{http_code}\n" http://localhost:8000/health/ 2>/dev/null || curl -s -o /dev/null -w "HTTP %{http_code}\n" http://localhost:8000/blog/
```

**Verificación:** Debe responder `HTTP 200`.

#### Paso 3.3 — Verificar URLs clave desde internet

```bash
curl -s -o /dev/null -w "HTTP %{http_code} %{url_effective}\n" -L https://jaimediaz.dev/
curl -s -o /dev/null -w "HTTP %{http_code} %{url_effective}\n" -L https://jaimediaz.dev/blog/
```

**Verificación:** Ambas deben responder `HTTP 200`.

## 🔍 Solución de problemas

### El servicio Gunicorn no arranca

```bash
# Ver logs
sudo journalctl -u jdiaz_gunicorn.service -n 50 --no-pager

# Ver errores de sintaxis de Python
python -c "import django; print('OK')"
```

### `collectstatic` falla

```bash
# Verificar que STATIC_ROOT existe en settings.py
grep -i static_root backend/jdsite/settings.py

# Forzar recolección
python manage.py collectstatic --noinput --clear
```

### La página devuelve 500

```bash
# Ver logs de Gunicorn
sudo journalctl -u jdiaz_gunicorn.service -n 100 --no-pager

# Ver logs de nginx (si aplica)
sudo tail -50 /var/log/nginx/error.log
```
