# HU-17.19 Filtros Avanzados Colapsables en Blog List

## Objetivo
Aplicar el mismo patrón de **filtros colapsables** implementado en el dashboard (HU-17.18) a la página pública de listado de artículos (`blog_list.html`), pero respetando los filtros específicos que ya existen en esta vista: búsqueda por texto, rango de fechas y categoría.

---

## 🔍 Análisis del estado actual

### Filtros existentes en `BlogListView` (backend)
| Parámetro GET | Backend                                                            | Frontend actual                   |
| ------------- | ------------------------------------------------------------------ | --------------------------------- |
| `q`           | title, content_html, description, slug, category__name, tags__name | Input `#blog-search-input`        |
| `category`    | category__slug                                                     | Sidebar (enlaces a `?category=X`) |
| `date_from`   | created_at__date__gte                                              | Flatpickr input `#date-from`      |
| `date_to`     | created_at__lt (date_to + 1 día)                                   | Flatpickr input `#date-to`        |
| `order`       | publish_date ASC/DESC                                              | Sidebar (enlaces a `?order=asc`)  |

### Estructura actual del template (blog_list.html)

```
┌─ FILTROS DE BÚSQUEDA ──────────────────────────────────────────────┐
│                                                                     │
│  Fila 1: [📅 Desde ▾] → [📅 Hasta ▾] [✓] [✕]  [✍ Escribir] [⚖ Moderación] │
│                                                                     │
│  Fila 2: [🔍 Buscar...                                ] [Buscar]    │
│          [🧹 Limpiar filtros]                                       │
│                                                                     │
│  Hint: "Mostrando resultados para: 'X' · Categoría: Y · Desde: Z"  │
└─────────────────────────────────────────────────────────────────────┘
```

### Problemas detectados
1. **Carga cognitiva alta**: El filtro de fechas (con Flatpickr y dos inputs) + botones de acción están siempre visibles, incluso cuando el usuario solo quiere buscar por texto.
2. **Hint textual pobre**: El `.blog-search-hint` actual es solo texto plano, no tiene chips interactivos con ✕ para eliminar filtros individuales.
3. **Sin sticky**: Al hacer scroll, los filtros desaparecen. El usuario debe volver arriba para cambiar la búsqueda.
4. **Sin contador de resultados**: No muestra cuántos artículos se encontraron.
5. **Sin mejoras UX en buscador**: No tiene botón X para limpiar, ni búsqueda en vivo con debounce, ni atajo Escape.

```
│ [🔍 "titulo" ✕] [📅 Desde: 2026-01-01 ✕] [📁 Categoría: Python ✕]   │
│ [🧹 Limpiar todo]                                                    │
└──────────────────────────────────────────────────────────────────────┘
```

### Principios aplicados (idénticos a HU-17.18)
1. **Progressive disclosure**: primero el buscador, lo avanzado bajo demanda
2. **Active filter chips siempre visibles**: el usuario nunca olvida qué filtros tiene
3. **Sticky**: fila 1 + chips se quedan fijos al hacer scroll
4. **Toggle con icono ⚙️ y texto**: indica que hay más opciones
5. **aria-expanded**: soporte de accesibilidad

## 📦 Contrato de comunicación backend ↔ frontend

| Parámetro GET | Valores          | Default  | Backend (BlogListView.get_queryset)                 | Frontend                                     |
| ------------- | ---------------- | -------- | --------------------------------------------------- | -------------------------------------------- |
| `q`           | string           | `""`     | `title__icontains`, `content_html__icontains`, etc. | Input `#blog-search-input` + toggle collapse |
| `category`    | string (slug)    | `""`     | `category__slug=valor`                              | Sidebar (no cambia) + chip                   |
| `date_from`   | `YYYY-MM-DD`     | `""`     | `created_at__date__gte=valor`                       | Flatpickr `#date-from` dentro del collapse   |
| `date_to`     | `YYYY-MM-DD`     | `""`     | `created_at__lt=valor+1día`                         | Flatpickr `#date-to` dentro del collapse     |
| `order`       | `"desc"`/`"asc"` | `"desc"` | `publish_date` DESC/ASC                             | Sidebar (no cambia)                          |

**⚠️ Diferencia clave con dashboard**: En `blog_list.html` NO hay filtros de `is_published` ni `moderation`. Los filtros avanzados son: **rango de fechas** y **categoría** (la categoría se controla desde el sidebar, pero debe mostrarse como chip).

### ⚠️ Regla estricta de separación

| Recurso        | Dónde va                    | Prohibido                                      |
| -------------- | --------------------------- | ---------------------------------------------- |
| **JavaScript** | `blog_list.js` (archivo)    | ❌ NO usar bloques `<script>` en el HTML        |
| **CSS**        | `blog_list.css` (archivo)   | ❌ NO usar `style=""` inline en el HTML         |
| **HTML**       | `blog_list.html` (template) | ✅ Solo estructura + template tags + clases CSS |

**Todo el comportamiento JS** (toggle, botón X, debounce, sticky observer, IntersectionObserver) debe ir en `blog_list.js`. No se debe inyectar JS dentro del HTML bajo ninguna circunstancia.

**Todos los estilos** (chips, sticky, botón X) deben ir en `blog_list.css`. No se debe usar `style="..."` en el HTML para nada relacionado con esta HU.

Agregar `has_active_filters` al `get_context_data()`:

```python
def get_context_data(self, **kwargs):
    context = super().get_context_data(**kwargs)
    # ... código existente ...
    
    # HU-17.19: Detectar si hay filtros avanzados activos
    date_from = self.request.GET.get("date_from", "").strip()
    date_to = self.request.GET.get("date_to", "").strip()
    category = self.request.GET.get("category", "").strip()
    
    context["has_active_filters"] = bool(
        date_from or date_to or category
    )
    
    return context
```

```javascript
// =============================================
// HU-17.19: Toggle collapse filtros avanzados
// =============================================
// Manejo manual (sin data-bs-toggle)
(function() {
    var toggleBtn = document.getElementById('toggle-advanced-filters');
    var collapseEl = document.getElementById('advanced-filters-collapse');
    if (!toggleBtn || !collapseEl) return;

    function updateToggleText(isExpanded) {
        if (isExpanded) {
            toggleBtn.innerHTML = '<i class="fas fa-times me-1"></i><span id="toggle-advanced-text">Cerrar filtros</span>';
            toggleBtn.setAttribute('aria-expanded', 'true');
        } else {
            toggleBtn.innerHTML = '<i class="fas fa-cog me-1"></i><span id="toggle-advanced-text">Filtros avanzados</span>';
            toggleBtn.setAttribute('aria-expanded', 'false');
        }
    }

    toggleBtn.addEventListener('click', function() {
        var isExpanded = collapseEl.classList.contains('show');
        if (isExpanded) {
            collapseEl.classList.remove('show');
            updateToggleText(false);
        } else {
            collapseEl.classList.add('show');
            updateToggleText(true);
        }
    });

    if (collapseEl.classList.contains('show')) {
        updateToggleText(true);
    }
})();

// HU-17.19: Mejoras al buscador #blog-search-input
// =============================================
(function() {
    var searchInput = document.getElementById('blog-search-input');
    if (!searchInput) return;

    var form = searchInput.closest('form');
    var debounceTimer;

    // 1. Botón X para limpiar
    var clearBtn = document.createElement('button');
    clearBtn.type = 'button';
    clearBtn.className = 'btn-clear-search';
    clearBtn.innerHTML = '<i class="fas fa-times-circle"></i>';
    clearBtn.title = 'Limpiar búsqueda';
    clearBtn.setAttribute('aria-label', 'Limpiar búsqueda');

    var inputGroup = searchInput.closest('.input-group');
    if (inputGroup) {
        inputGroup.appendChild(clearBtn);
    }

    function toggleClearBtn() {
        clearBtn.style.display = searchInput.value.trim() ? 'block' : 'none';
    }
    toggleClearBtn();
    searchInput.addEventListener('input', toggleClearBtn);

    clearBtn.addEventListener('click', function() {
        searchInput.value = '';
        toggleClearBtn();
        searchInput.focus();
        var url = new URL(window.location.href);
        url.searchParams.delete('q');
        url.searchParams.delete('page');
        window.location.href = url.toString();
    });

    // 2. Búsqueda en vivo con debounce (500ms)
    searchInput.addEventListener('input', function() {
        clearTimeout(debounceTimer);
        var val = this.value.trim();
        debounceTimer = setTimeout(function() {
            if (val.length >= 2 || val.length === 0) {
                form.submit();
            }
        }, 500);
    });

    // 3. Atajo Escape
    searchInput.addEventListener('keydown', function(e) {
        if (e.key === 'Escape') {
            searchInput.value = '';
            toggleClearBtn();
            searchInput.blur();
        }
    });
})();

        );
        // Observar el siguiente elemento después del sticky
        var nextEl = stickyRow.nextElementSibling;
        if (nextEl) observer.observe(nextEl);
    }

    if (chipsRow) {
        var observer2 = new IntersectionObserver(
            function(entries) {
                entries.forEach(function(entry) {
                    if (!entry.isIntersecting) {
                        chipsRow.classList.add('is-sticky');
                    } else {
                        chipsRow.classList.remove('is-sticky');
                    }
                });
            },
            { threshold: 0 }
        );
        var nextEl2 = chipsRow.nextElementSibling;
        if (nextEl2) observer2.observe(nextEl2);
    }
})();
```
